## 2026-05-03 - Accessible Hidden Overlays & Redundant Screen Reader Text
**Learning:** In dark-theme designs, hover overlays (using opacity-0/opacity-100) on interactive elements remain inaccessible to keyboard users unless explicitly given `focus-visible:opacity-100` along with standard focus rings. Additionally, when wrapping visual text and icons within an interactive container that has an aria-label, the inner elements should be given `aria-hidden='true'` to avoid redundant and confusing screen reader announcements.
**Action:** When creating visually hidden interactive elements, always pair `opacity-0` with `focus-visible:opacity-100` and ensure focus rings are visible. Apply `aria-hidden='true'` to inner visible text/icons if the parent has an explicitly descriptive `aria-label`.
