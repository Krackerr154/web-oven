## 2024-04-24 - Interactive Div Keyboard Accessibility in Dark Theme
**Learning:** When making non-native elements (like `div` cards) interactive in a dark-theme UI (e.g., `web-oven`), it's crucial to ensure keyboard accessibility. Visually hidden hover overlays (like details panels) must include `focus-visible:opacity-100` alongside standard focus rings to ensure they are visible to keyboard users.
**Action:** Always add `role="button"`, `tabIndex={0}`, an `onKeyDown` handler (listening for 'Enter' and 'Space' and calling `e.preventDefault()`), and explicit focus rings (`focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-orange-500`) to interactive `div` elements. For hover-based overlays, conditionally apply `focus-visible:opacity-100` so keyboard navigation functions parity with mouse hover.
