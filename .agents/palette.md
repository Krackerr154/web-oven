## 2024-03-09 - Accessibility and Focus States on Custom Toast Components
**Learning:** Found that custom `react-hot-toast` style components or custom toast wrappers often lack accessibility features for their close buttons. The dismiss button is often an icon-only button that needs an explicit `aria-label` to be screen-reader friendly. Furthermore, these buttons need `focus-visible` styles against the dark slate background, otherwise keyboard users cannot see which element has focus.
**Action:** When creating or reviewing custom alert/toast components in web-oven, always ensure the close/dismiss button has an explicit `aria-label` (e.g., `aria-label="Dismiss toast"`) and highly visible focus styles using the established design system accent color (`focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-orange-500 rounded-sm`). This should be standard practice for all icon-only interactive elements.
