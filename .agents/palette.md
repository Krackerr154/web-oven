## 2024-05-17 - Accessible Interactive Cards and Hidden Overlays
**Learning:** When making non-native elements (like `div` cards) interactive in web-oven, adding `role="button"` and `tabIndex={0}` is required, but ensuring 'Enter' and 'Space' trigger the action requires an explicit `onKeyDown` handler with `e.preventDefault()` to prevent page scrolling. Furthermore, for interactive elements placed inside visually hidden overlays (e.g., hidden via `opacity-0` with absolute positioning instead of `display: none`), the inner elements will still receive keyboard focus. We must dynamically manage focusability using `tabIndex={isVisible ? 0 : -1}` to prevent screen reader and keyboard users from interacting with invisible UI.
**Action:** Always test interactive cards and complex hover overlays with keyboard navigation. Dynamically apply `tabIndex=-1` to any buttons within `opacity-0` containers, and always remember the `e.preventDefault()` on custom `onKeyDown` handlers for 'Space'.
